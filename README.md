# 🏢 biz-game — геймифицированный дашборд бизнеса на реальных данных 1С

> Пиксельный офис «вид сверху», где **каждое событие — это реальные данные из 1С/Bitrix24**.
> Менеджеры ходят к полкам и собирают заказы, ИИ-агенты-собаки приносят отчёты,
> а сбоку — живая панель KPI, выручки, людей и задач собственника.

*A top-down pixel-art office that visualizes a real business in real time: every order,
manager and KPI is pulled live from 1C (ERP) / Bitrix24. Built with Phaser 3 + Vite + TypeScript.*

---

## Что это

Это «играбельный» дашборд для собственника бизнеса (на примере дистрибуции косметики на
**1С: Управление торговлей 11.5**). Вместо скучных графиков — живой офис:

- 🧑‍💼 **Менеджеры** сидят за столами; на каждый реальный заказ один встаёт, идёт к стеллажу,
  собирает товар и несёт на стол сборки.
- 🐕 **ИИ-агенты — собаки** (Геннадий, Алёша, Жорик, Степаныч): анализ менеджеров, ключевые
  клиенты, прогноз продаж, возврат «спящих» клиентов — с **реальными цифрами результата за месяц**.
- 📊 **Боковая панель** (HTML-оверлей): Обзор, Выручка (по периодам + график), Люди (рейтинг
  менеджеров), Задания (брифинг собственника из реальных данных), Агенты.
- 🎉 **Праздник продажи**: на новый заказ — конфетти и баннер «НОВАЯ ПРОДАЖА» в стиле старых игр.
- 🟢 **Честность данных**: всё помечено «1С» (реальное) или «демо»; есть индикатор свежести снапшота.

## Как это работает (архитектура)

```
  1С OData (только чтение)
        │   systemd timer / cron, каждые 30 мин
        ▼
  scripts/build_snapshot.py  ──►  real.json  ──►  nginx (статика)
                                                      │  игра опрашивает раз в 5 мин (no-store)
                                                      ▼
                            Phaser-сцена + HTML-панель  ◄── buildModel() формирует модель дашборда
```

Главный приём — **развязка «данные ↔ сцена»**: бэкенд лишь кладёт `real.json`, фронтенд его
читает и оживляет. Никаких секретов в коде; 1С дёргается только на чтение и не нагружается
(агрегация на стороне скрипта).

## Стек

- **Phaser 3.90** — рендер офиса, спрайты, анимации, камера
- **Vite 5 + TypeScript** — сборка и dev-сервер
- **GSAP** — анимации панели (счётчики, появление, баннеры)
- **Python 3 (stdlib)** — снапшот из 1С OData (без зависимостей)

## Быстрый старт (локально)

```bash
cd app
npm install
npm run dev          # http://localhost:5600
```

Без `real.json` дашборд работает в демо-режиме (показывает заглушки). Чтобы увидеть данные —
сгенерируйте снапшот из своей 1С (см. ниже) или положите свой `app/public/assets/data/real.json`.

Сборка прода: `npm run build` → `app/dist`.

## Подключение к своей 1С

```bash
cd scripts
cp .env.example .env      # вписать ODATA_URL / ODATA_USER / ODATA_PASS (пользователь 1С — только чтение!)
chmod 600 .env
python3 build_snapshot.py # сгенерирует real.json
```

Подробности пайплайна, деплой на сервер (nginx + systemd timer) и состав `real.json` —
в [`scripts/README.md`](scripts/README.md).

## Структура

```
app/                     Phaser-приложение (Vite + TS)
  src/scenes/            BootScene (загрузка), OfficeScene (сцена офиса)
  src/ui/GameUI.ts       HTML-панель: вкладки, KPI, анимации, праздник продажи
  src/data/              Dashboard.ts (модель), Briefing.ts (задачи собственника)
  public/assets/         тайлы офиса (своя графика) + данные
tools/gen_office_tiles.mjs  процедурный генератор части пиксель-ассетов
scripts/                 build_snapshot.py (1С→real.json) + деплой (systemd/nginx)
```
